## ğŸ’¡ Developer Guidelines & Tips
*Rules I follow to ensure my code stays clean and professional:*

> [!TIP]
> **Consistency > Intensity:** 1 hour of coding every day is better than 10 hours once a week.

1.  **DRY (Don't Repeat Yourself):** If I write the same code twice, I turn it into a function.
2.  **KISS (Keep It Simple, Stupid):** Simple code is easier to debug and maintain than "clever" code.
3.  **Read the Docs:** Before watching a 2-hour tutorial, I spend 15 minutes reading the official documentation.
4.  **Comment the "Why," not the "What":** I write comments to explain the logic, not just to describe the syntax.
5.  **Commit Often:** I use Git to save my progress like a "save game" in an RPG.
